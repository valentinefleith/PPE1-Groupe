Description du projet

Nous réalisons ce projet dans le cadre du cours de Programmation et Projet Encadré du Master 1 Traitement Automatique des Langues (Sorbonne nouvelle, Paris Nanterre et INALCO). Le but de ce projet est de constituer un corpus de textes à partir d'aspirations de pages internet dans 3 langues différentes, puis, à partir de ce corpus, d'étudier le comportement d'un mot sur les différentes pages web. Cela nous permet d'interpréter la façon dont le mot est perçu dans chaque langue, en fonction notamment de ses cooccurrents.

Choix du mot

Les trois langues que nous allons étudier sont le français, l'anglais et le chinois (simplifié). Nous nous sommes mises d'accord pour choisir le mot "argent" en français, "money" en anglais, et "钱" en chinois pour notre projet. Nous avons choisi ce mot car nous sommes intéressées de découvrir ses utilisations et ses emplois sur les pages web dans différentes langues. Ce choix s'est porté sur ces termes car ils représentent un concept universel et sont susceptibles d'être fréquemment utilisés dans une grande variété de contextes sur le web. Comprendre comment le concept d'argent est abordé dans différentes langues peut révéler des perspectives variées sur l'économie mondiale, les inégalités, et les politiques monétaires. Les questions principales qui ont guidé notre choix sont les suivantes :

-> Ce mot est-il majoritairement utilisé par des économistes ? Ou bien ces derniers emploient-ils des mots plus précis ?
-> Y a-t-il est sentiments qui gravitent autour du mot "argent" ? Lesquels ?
-> Comment la vision de l'argent diffère-t-elle d'un pays à l'autre ? (De plus, en français, ce mot a un homonyme. Nous cherchons donc également à comprendre comment la perception de l'argent diffère d'un pays à l'autre, ce qui ajoute une couche supplémentaire de complexité dans son interprétation et son usage.)

Pour chaque langues, on a trouvé environ 50 pages web qui contiennent le mot "argent" sur des plateformes variées tel que les sites d'actualités, les blogs, les sites de bourse, etc.

Composition du corpus

Nous avons donc constitué un corpus par mot. Le corpus en français a été constitué par Ashley et comporte 50 URLS. Le corpus en anglais a été constitué par Valentine et comporte 44 URLS. Certains URLs ne fonctionnent pas dans le tableau, mais nous avons conservé les colonnes pour montrer que le script arrête le traitement des pages dont le code HTTP indique une erreur. Enfin, le corpus en chinois a été recueilli par Lidan, et comporte 54 URLs. La aussi, certains ne fonctionnent pas mais apparaissent tout de même dans le tableau.

Traitement du corpus

Nous avons réalisé tous les traitements sur nos 3 corpus à l'aide de scripts bash écrits durant les cours. Les scripts ainsi que des explications relatives à chacun d'entre eux sont retrouvables ici (création des tableaux), ici (concordancier) et ici (script pour rendre le corpus traitable par Itrameur). Pour mettre les scripts sur le site, nous avons utilisé une librairie JS qui s'appelle prism afin d'avoir des couleurs. Toutefois, comme nous avions des balises html au sein de notre script bash, nous avons dû écrire un script python pour formater correctement les scripts en enlevant les caractères spécifiques au HTML qui pourraient être interprétés comme des marques de vraies balises. Ce script est retrouvable sur notre github.
